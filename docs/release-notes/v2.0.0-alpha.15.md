# v2.0.0-alpha.15 release notes

## PR Title
`feat: phase-visibility and rendering hardening for alpha.15`

## PR Description
This PR targets the `v2.0.0-alpha.15` ship-gate with three priorities: make visible rendering progress reproducible again (`alpha.8` → `alpha.9`), stabilize feature rollout defaults, and add observability/perf safeguards for production viability.

### Session summary
- **Total commits this session:** 2 (`v2.0.0-alpha.14` → `v2.0.0-alpha.15`)
- **Alpha.15 included commits:** `cb84b05`, `4972bb3`

### Key visible-rendering breakthroughs
- **alpha.8 — visible baseline restored** (`d35f5c8`): closed the initial visibility dead-end with corrected 3D world draw paths so core voxel/building actors were again visible.
- **alpha.9 — limbed visibility pass stabilized** (`96ca036`): completed the alpha.8 → alpha.9 follow-through with limbed rendering correctness and visual persistence fixes.
- **Transpiler guard hardening:** captured in `ADR-0016`; 12 explicit transpiler guards were introduced to keep world projection, velocity/effect, and sprite transforms from drifting into unsupported code paths.

### Default-flag cascade
- Applied the phase ship-gate pattern from `ADR-0005`: phase flags remain disabled by default until a phase has completed smoke-test validation, then their specific gate flips on.
- This alpha keeps new renderer behavior behind phase defaults and avoids broad opt-in flips that could regress startup behavior.

### Perf telemetry
- Strengthened runtime signal paths for rendering and mesh/instance flows, including cache metrics and frame-level telemetry used by AutoTest/hand-off diagnostics.
- Added/retained counters that expose render path shape and cache behavior so regressions are visible before manual smoke checks.

### LRU cache work
- Continued LRU alignment across rendering helpers with bounded eviction behavior and deferred cleanup semantics.
- Reinforced cache lifecycle assumptions used by `ProcGen`/impostor/building-ish pipelines to prevent silent memory growth while preserving visible cache hit behavior.

### Transpiler guards
- Guarded risky IL hooks and entrypoints in the alpha.8/alpha.9 chain to prevent patch misapplication and silent fallback behavior in non-targeted code.
- This hardens both render correctness and performance by avoiding unpredictable branch behavior when world-mode state changes.
