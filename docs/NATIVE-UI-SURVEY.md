# Native UI Survey

| Option | Feasibility from net48 Mono mod | Cross-platform win/mac/linux | Effort estimate | What we lose |
|---|---|---|---|---|
| Tauri sidecar | Low to medium. Works only as an external companion app; the mod can talk to it over IPC, but it is awkward from a net48 Mono game runtime. | Yes, in principle. Tauri targets desktop OSes, but packaging and launch integration vary by platform. | High. Separate process, IPC protocol, packaging, and lifecycle management. | Tight in-game integration, simple distribution, and a lot of the "just ship a mod" convenience. |
| Avalonia in-process | Medium. The UI code can live in the mod or a helper assembly, but running a modern XAML stack inside a net48 Mono mod is a compatibility and deployment risk. | Yes. Avalonia is the strongest cross-platform option on paper. | Medium to high. Still need host integration, rendering hookup, and careful dependency management. | Native WorldBox look and feel, plus some runtime simplicity if the host cannot load the stack cleanly. |
| WinUI3 P-Invoke | Low. This is effectively Windows-only and not a natural fit for a net48 Mono mod boundary. | No. Windows only. | High. Interop, threading, COM-style hosting concerns, and Windows app-model complexity. | Cross-platform support and most of the safety of staying inside the game process. |
| Stay in-game | High. This is the path that fits the current mod architecture best and avoids runtime mismatch. | Yes, to the extent the game itself is cross-platform. | Low to medium. Reuses existing rendering, input, and save-state plumbing. | We lose rich native controls, OS-level window chrome, and the ability to build a separate tool-like experience. |

Recommendation: stay in-game unless there is a hard requirement for a separate desktop tool. It is the lowest-risk option for a net48 Mono mod, preserves cross-platform behavior, and avoids introducing a second app lifecycle, packaging pipeline, and IPC surface. If a separate UI becomes necessary later, Avalonia is the most credible general-purpose fallback; Tauri is better when the UI is mostly a web-based companion, and WinUI3 should be treated as Windows-only and last-resort.
